R语言中的面向对象编程

发表于 2017-04-02 | 分类于 R语言

我们都知道面向对象的三大特性：封装，继承，多态。虽然R语言和我们熟悉的面向对象的语言（如c++，java，python）不同，但是R其实也是一个面向对象的语言。只是由于为了降低R的初始学习难度，所以开发者将很多函数都封装起来便于初学者使用。

s3类

事实上，大多数R中内置的类都是S3类。一个S3类包含一个列表，再附加一个类名属性和调度的功能。而后者为R中泛型函数的使用提供了基础。而S4类是后来为了增加安全性才开发出来的。
R中的多态是指同一个函数可以针对不同的类调用不同的操作。例如plot函数，对许多不同的数据类型的对象，画出的图类型也不同。还有很多函数，如summary和print函数都是如此。
S3类中，一个类的实例是通过构建一个列表的方式来创建的，这个列表的组件是该类的成员变量。类的属性通过attr()或者class()函数手动设置，然后再定义各种泛型函数的实现方法。

由继承的目的性能够知道，打印college_class将会调用print()函数，R会去查找college_student类的打印方法，这是因为college_student是college_class的两个类名中的第一个，当没有找到其对应的方法时，R会尝试查找另一个类student的打印方法。

S4类

一些R开发者认为s3类不具有面向对象编程固有的安全性。例如，’student’类中有三个字段：name，age，sex。下面是一些常见的错误：
1忘记输入age的值
2将sex错拼为sax
等等，以上这些错误R都不会自动提醒。而S4类的目标是产生自动提醒从而避免这些错误的发生。
R中使用setClass()来定义一个S4类。现在来定义一个新的’person’类。

能够看到定义的新类中有三个成员变量，而且每个变量都有明确的类型。然后使用new()函数为这个类创建一个实例，joe。
打印joe后能够得到以下结果：

joe
An object of class “person”
Slot “name”:
[1] “will”

Slot “salary”:
[1] 50000

Slot “union”:
[1] TRUE

需要注意的是，此处成员变量称为slot，通过@符号引用。例如：
>joe@salary
打印结果为：

[1]50000

同时也可以使用slot()函数来查看，
>slot(joe,'salary')
打印结果为：

[1]50000

用类似的方法也可以给成员变量复制，例如:
joe@salary <- 65000

由此能够看到S4类相比S3类更具有安全性。S3类的对象不会有任何报错信息，因为S3类仅仅是列表，所以能够添加新的组件，不管你是有意还是无意的。
