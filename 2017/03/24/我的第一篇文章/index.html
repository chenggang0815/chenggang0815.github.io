记一下搭建过程中踩过的坑

发表于 2017-03-24 | 分类于 博客搭建

惯例：Hello World

搭建环境

测试系统：ubuntu16.04
当然还需要安装：nodejs，hexo，git
耗时：两天

搭建步骤

第一天

1.首先在github上新建一个repository，具体细节没什么好说的，网上的教程有很多。
2.安装nodejs
sudo add-apt-repository ppa:chris-lea/node.js
sudo apt-get update
sudo apt-get install nodejs
3.安装git
sudo apt-get install git
4.安装hexo
sudo npm install hexo-cli -g
当然除了hexo其他我原来都安装过了。

坑一

官网上的安装命令为:npm install hexo-cli -g,但是没有root权限会报错

坑二

我在安装好nodejs和npm后，安装hexo一直报错，我在经过了更改文件夹权限，重装nodejs等操作后还是无法成功安装，后来仔细看error，发现少了一个依赖包…
于是又 sudo apt install nodejs-legacy，安装成功。
前面的这些操作，包括阅读hexo的概念，理解整个搭建框架用了我一天时间,(其实睡了大半天

第二天

1.安装好hexo之后，熟悉了它的基本命令，开始搭建主题，菜单，社交链接等(都是一些小坑…
虽然还有很多包括分类等功能还没做好…
2.去万网上买了个域名，然后关联到github

坑三

由于每次测试都要输入github账号，密码，所以直接关联了ssh。关联过程中我找到的最靠谱的方法就是把_config.yml中的repo改为：
repo: git@github.com:chenggang0815/chenggang0815.github.io.git
一次性成功…

最后

总结一下，虽然弄了两天，但除去中间睡觉和找同学聊人生理想的时间…应该是一天的工作量。虽然遇到了一些坑，但最后终于可以愉快的写东西了。
后续会边完善功能边写文章(主要是自己在工作上的一些学习和感悟)，希望自己不要忘记搭建博客的初心，可以持续输出高质量的东西。

最后的最后

我现在迫切需要复习一下markdown语法，感觉排版惨不忍睹..(逃
